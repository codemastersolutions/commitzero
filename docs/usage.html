Usage Guide

Learn how to use CommitZero CLI commands, validate commits, and integrate with your development workflow.

CLI Commands

CommitZero provides several commands to help you manage commit validation and Git hooks.

Initialize Project

Setup
commitzero init

Creates configuration file and installs Git hooks for automatic validation.

What it does:
  • Creates .commitzero.json configuration file
  • Installs pre-commit and commit-msg Git hooks
  • Backs up existing hooks if present

Validate Commit Message

Validation
commitzero validate "feat: add user authentication"
commitzero validate --file commit-message.txt

Validates a commit message against your configuration rules.

Manage Git Hooks

Hooks

Install Hooks

commitzero install

Uninstall Hooks

commitzero uninstall

Manually install or remove Git hooks for commit validation.

Get Help

Info
commitzero --help
commitzero --version

Display help information and version details.

Commit Validation

Understanding how CommitZero validates your commit messages and what makes a valid commit.

✅ Valid Commit Messages

feat: add user authentication system
fix(api): resolve login endpoint timeout issue
docs: update installation instructions
feat!: change API response format

❌ Invalid Commit Messages

added new feature

Missing type prefix

feat:

Missing description

feat: Add New Feature

Description should start with lowercase

unknown: add something

Invalid commit type

Validation Rules

Format Requirements

  • Must start with valid type
  • Colon and space after type
  • Description starts lowercase
  • No period at the end

Length Limits

  • Subject: 1-100 characters
  • Body: max 500 characters
  • Footer: max 100 characters

Git Hooks Integration

CommitZero automatically validates commits through Git hooks, ensuring consistency across your team.

Installed Hooks

Pre-commit Hook

  • Runs before commit creation
  • Executes pre-commit commands
  • Can run linting, tests, etc.

Commit-msg Hook

  • Validates commit message format
  • Checks against configuration
  • Prevents invalid commits

Workflow Integration

Integrate CommitZero with your existing development workflow and tools.

CI/CD Integration

Validate commit messages in your CI/CD pipeline: 

# GitHub Actions example
name: Validate Commits
on: [push, pull_request]
jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
        with:
          fetch-depth: 0
      - uses: actions/setup-node@v3
        with:
          node-version: '18'
      - run: npm install -g commitzero
      - run: commitzero validate-range HEAD~1..HEAD

Troubleshooting

Common issues and their solutions when using CommitZero.

Common Issues

Hooks not running

If Git hooks aren't executing, check:

  • Hook files exist in .git/hooks/
  • Hook files are executable (chmod +x)
  • No syntax errors in hook scripts
commitzero install --force

Configuration not found

If CommitZero can't find your configuration:

  • Ensure .commitzero.json exists in project root
  • Check JSON syntax is valid
  • Use --config flag to specify custom path
commitzero validate --config ./custom-config.json

Permission denied errors

On Unix systems, you might need to make hooks executable:

chmod +x .git/hooks/pre-commit .git/hooks/commit-msg

Debug Mode

Enable verbose logging to troubleshoot issues:

commitzero validate "feat: test" --verbose

This will show detailed validation steps and configuration loading.

Best Practices

Follow these best practices to get the most out of CommitZero.

Writing Good Commit Messages

✅ Do

  • Use imperative mood ("add" not "added")
  • Be specific and descriptive
  • Use scopes for context
  • Keep subject line under 50 chars when possible
  • Use body for detailed explanations

❌ Don't

  • Use vague descriptions ("fix stuff")
  • Mix multiple unrelated changes
  • Use past tense ("added feature")
  • End with periods
  • Capitalize first letter of description

Team Guidelines

Establish Clear Rules

  • Define project-specific commit types and scopes
  • Document commit message conventions in README
  • Set up shared configuration file
  • Train team members on conventional commits

Automate Enforcement

  • Install hooks automatically in setup scripts
  • Add CommitZero to CI/CD pipeline
  • Use pre-commit commands for code quality
  • Block non-compliant commits in pull requests
