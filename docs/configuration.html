Configuration

Customize CommitZero to fit your project's needs. Configure commit types, scopes, validation rules, and Git hooks integration.

Getting Started

Initialize CommitZero in your project to create a default configuration file and set up Git hooks.

Initialize Configuration

commitzero init

This command creates a .commitzero.json configuration file and installs Git hooks for automatic validation.

What happens during initialization?

  • • Creates .commitzero.json with default configuration
  • • Installs pre-commit Git hook for validation
  • • Sets up commit-msg hook for message validation
  • • Creates backup of existing hooks (if any)

Configuration File

The .commitzero.json file contains all configuration options for your project. Here's the default structure:

Default Configuration

{
  "types": [
    { "type": "feat", "description": "A new feature" },
    { "type": "fix", "description": "A bug fix" },
    { "type": "docs", "description": "Documentation only changes" },
    { "type": "style", "description": "Changes that do not affect the meaning of the code" },
    { "type": "refactor", "description": "A code change that neither fixes a bug nor adds a feature" },
    { "type": "perf", "description": "A code change that improves performance" },
    { "type": "test", "description": "Adding missing tests or correcting existing tests" },
    { "type": "build", "description": "Changes that affect the build system or external dependencies" },
    { "type": "ci", "description": "Changes to our CI configuration files and scripts" },
    { "type": "chore", "description": "Other changes that don't modify src or test files" },
    { "type": "revert", "description": "Reverts a previous commit" }
  ],
  "scopes": [],
  "subjectMaxLength": 100,
  "subjectMinLength": 1,
  "allowBreakingChanges": true,
  "allowCustomScopes": true,
  "footerMaxLength": 100,
  "bodyMaxLength": 500,
  "preCommitCommand": null,
  "hookPath": ".git/hooks",
  "language": "en"
}

Commit Types

Define the allowed commit types for your project. Each type should have a clear purpose and description.

Standard Types

feat New features
fix Bug fixes
docs Documentation
style Code style changes
refactor Code refactoring

Custom Types

Add your own commit types:

{
  "types": [
    {
      "type": "security", 
      "description": "Security improvements"
    },
    {
      "type": "deps", 
      "description": "Dependency updates"
    },
    {
      "type": "config", 
      "description": "Configuration changes"
    }
  ]
}

Scopes

Scopes provide additional context about what part of the codebase is affected by the commit.

Define Project Scopes

Configure scopes that match your project structure: 

{
  "scopes": [
    "api",
    "ui",
    "auth",
    "database",
    "config",
    "tests",
    "docs"
  ],
  "allowCustomScopes": true
}

Tip: Set allowCustomScopes: true to allow developers to use scopes not in the predefined list.

Scope Usage Examples

feat(api): add user authentication endpoint
fix(ui): resolve button alignment issue
docs(config): update installation instructions

Validation Rules

Configure validation rules to enforce consistent commit message formatting.

Subject Line Rules

"subjectMaxLength": 100
"subjectMinLength": 1

Body & Footer Rules

"bodyMaxLength": 500
"footerMaxLength": 100

Breaking Changes

"allowBreakingChanges": true

When enabled, allows commits with ! after type/scope or BREAKING CHANGE: in footer.

Language Settings

"language": "en" // en, pt, es

Supported languages: English (en), Portuguese (pt), Spanish (es).

Git Hooks Integration

Configure Git hooks for automatic commit validation and pre-commit commands.

Hook Management Commands

Install Hooks

commitzero install

Uninstall Hooks

commitzero uninstall

Pre-commit Commands

Run custom commands before each commit (e.g., linting, testing): 

{
  "preCommitCommand": "npm run lint && npm test"
}

Note: If the pre-commit command fails, the commit will be aborted.

Custom Hook Path

Specify a custom path for Git hooks (useful for monorepos or custom setups): 

{
  "hookPath": ".git/hooks"  // Default
  // or
  "hookPath": ".githooks"   // Custom path
}

Advanced Configuration

Advanced configuration options for complex project setups.

Environment-specific Configuration

Use different configurations for different environments:

Development

.commitzero.dev.json

Production

.commitzero.prod.json

Usage

commitzero --config .commitzero.dev.json

Monorepo Configuration

Configure scopes based on package structure: 

{
  "scopes": [
    "packages/ui",
    "packages/api",
    "packages/shared",
    "apps/web",
    "apps/mobile",
    "tools",
    "docs"
  ],
  "allowCustomScopes": false,
  "subjectMaxLength": 120
}
