API Reference

Complete programmatic interface for CommitZero - integrate commit validation into your applications and tools.

Overview

CommitZero provides a comprehensive JavaScript/TypeScript API for programmatic access to commit validation, configuration management, and Git hooks integration. This allows you to integrate CommitZero functionality into your own tools, CI/CD pipelines, and applications.

Key Features

  • Validate commit messages programmatically
  • Parse and format commit messages
  • Manage configuration files
  • Install and manage Git hooks
  • Full TypeScript support with type definitions

Installation

Install CommitZero as a dependency in your project to access the programmatic API:

NPM 
npm install commitzero
Yarn 
yarn add commitzero
Basic Import 
import { validate, parse, format } from 'commitzero';

// Or using CommonJS
const { validate, parse, format } = require('commitzero');

Core API

The core API provides functions for validating, parsing, and formatting commit messages.

validate(message, options?)

Validates a commit message against Conventional Commits standards and custom rules.

Parameters

Parameter Type Description
message string The commit message to validate
options ValidationOptions? Optional validation configuration
Example Usage 
import { validate } from 'commitzero';

const result = await validate('feat: add user authentication');

if (result.valid) {
  console.log('✅ Valid commit message');
} else {
  console.log('❌ Invalid commit message');
  result.errors.forEach(error => {
    console.log(`- ${error.message}`);
  });
}

Return Value

Returns a ValidationResult object:

interface ValidationResult {
  valid: boolean;
  errors: ValidationError[];
  warnings: ValidationWarning[];
  parsed?: ParsedCommit;
}

parse(message)

Parses a commit message into its constituent parts (type, scope, description, body, footer).

Example Usage 
import { parse } from 'commitzero';

const parsed = parse('feat(auth): add OAuth2 integration

Implement OAuth2 authentication with Google and GitHub providers.

Closes #123');

console.log(parsed);
// {
//   type: 'feat',
//   scope: 'auth',
//   description: 'add OAuth2 integration',
//   body: 'Implement OAuth2 authentication with Google and GitHub providers.',
//   footer: 'Closes #123'
// }

format(commit)

Formats a parsed commit object back into a conventional commit message string.

Example Usage 
import { format } from 'commitzero';

const commit = {
  type: 'feat',
  scope: 'auth',
  description: 'add OAuth2 integration',
  body: 'Implement OAuth2 authentication with Google and GitHub providers.',
  footer: 'Closes #123'
};

const formatted = format(commit);
console.log(formatted);
// 'feat(auth): add OAuth2 integration
//
// Implement OAuth2 authentication with Google and GitHub providers.
//
// Closes #123'

Configuration API

Functions for loading and managing CommitZero configuration files.

loadConfig(path?)

Loads configuration from a file or searches for configuration in standard locations.

Example Usage 
import { loadConfig } from 'commitzero';

// Load from default locations
const config = await loadConfig();

// Load from specific path
const customConfig = await loadConfig('./custom-commitzero.config.js');

console.log(config.types); // Available commit types
console.log(config.scopes); // Available scopes

createConfig(options)

Creates a new configuration file with the specified options.

Example Usage 
import { createConfig } from 'commitzero';

await createConfig({
  path: './commitzero.config.js',
  types: ['feat', 'fix', 'docs', 'style', 'refactor'],
  scopes: ['api', 'ui', 'auth', 'db'],
  rules: {
    'subject-case': ['error', 'lower-case'],
    'subject-max-length': ['error', 50]
  }
});

Hooks API

Functions for managing Git hooks integration.

installHooks(options?)

Installs Git hooks for automatic commit message validation.

Example Usage 
import { installHooks } from 'commitzero';

// Install with default options
await installHooks();

// Install with custom options
await installHooks({
  hooks: ['commit-msg', 'pre-commit'],
  gitDir: './.git',
      force: true
});

Note: installHooks respects Git's core.hooksPath. The CLI commitzero install-hooks sets core.hooksPath to .commitzero/hooks so hooks are version-controlled; otherwise the default .git/hooks is used. Set force: true to overwrite existing managed blocks or update hooks configuration.

uninstallHooks(options?)

Removes CommitZero Git hooks from the repository.

Example Usage 
import { uninstallHooks } from 'commitzero';

// Uninstall all hooks
await uninstallHooks();

// Uninstall specific hooks
await uninstallHooks({
  hooks: ['commit-msg'],
  gitDir: './.git'
});

Note: uninstallHooks removes only CommitZero-managed blocks from hook files and preserves other content. It does not remove package.json scripts; the scripts commitzero, commitzero:install, and commitzero:uninstall are preserved.

TypeScript Types

CommitZero includes comprehensive TypeScript type definitions for all API functions and data structures.

Core Types 
interface ParsedCommit {
  type: string;
  scope?: string;
  description: string;
  body?: string;
  footer?: string;
  breaking?: boolean;
}

interface ValidationResult {
  valid: boolean;
  errors: ValidationError[];
  warnings: ValidationWarning[];
  parsed?: ParsedCommit;
}

interface ValidationError {
  message: string;
  rule: string;
  level: 'error' | 'warning';
}
Configuration Types 
interface CommitZeroConfig {
  types: string[];
  scopes?: string[];
  rules: Record;
  parserPreset?: string;
  formatter?: string;
  helpUrl?: string;
}

interface ValidationOptions {
  config?: CommitZeroConfig;
  cwd?: string;
  from?: string;
  to?: string;
}

Complete Examples

Custom Validation Tool

validate-commits.js 
#!/usr/bin/env node

import { validate, loadConfig } from 'commitzero';
import { readFileSync } from 'fs';
import { execSync } from 'child_process';

async function validateCommits() {
  try {
    // Load configuration
    const config = await loadConfig();
    
    // Get commit messages from git log
    const commits = execSync('git log --oneline -10 --pretty=format:%s')
      .toString()
      .split('\n')
      .filter(Boolean);

    let hasErrors = false;

    for (const commit of commits) {
      const result = await validate(commit, { config });
      
      if (!result.valid) {
        console.log(`❌ ${commit}`);
        result.errors.forEach(error => {
          console.log(`   ${error.message}`);
        });
        hasErrors = true;
      } else {
        console.log(`✅ ${commit}`);
      }
    }

    if (hasErrors) {
      process.exit(1);
    }
  } catch (error) {
    console.error('Error validating commits:', error.message);
    process.exit(1);
  }
}

validateCommits();

CI/CD Integration

GitHub Actions Workflow 
name: Validate Commits

on:
  pull_request:
    branches: [main]

jobs:
  validate-commits:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
        with:
          fetch-depth: 0
      
      - uses: actions/setup-node@v3
        with:
          node-version: '18'
      
      - run: npm install commitzero
      
      - name: Validate commit messages
        run: |
          node -e "
          import { validate } from 'commitzero';
          import { execSync } from 'child_process';
          
          const commits = execSync('git log origin/main..HEAD --pretty=format:%s')
            .toString().split('\n').filter(Boolean);
          
          let hasErrors = false;
          for (const commit of commits) {
            const result = await validate(commit);
            if (!result.valid) {
              console.log('❌', commit);
              result.errors.forEach(e => console.log('  ', e.message));
              hasErrors = true;
            }
          }
          if (hasErrors) process.exit(1);
          "
