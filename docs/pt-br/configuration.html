Configuração

Configure CommitZero para atender às necessidades específicas do seu projeto e equipe

Primeiros Passos

CommitZero pode ser configurado através de um arquivo de configuração no seu projeto. Para criar uma configuração inicial, execute: 

commitzero init

Isso criará um arquivo .commitzero.json na raiz do seu projeto com configurações padrão.

Arquivo de Configuração

CommitZero procura por arquivos de configuração na seguinte ordem:

  • .commitzero.json
  • .commitzero.js
  • commitzero.config.js
  • Seção commitzero no package.json

Estrutura Básica

{
  "types": {
    "feat": {
      "description": "Uma nova funcionalidade",
      "emoji": "✨"
    },
    "fix": {
      "description": "Correção de bug",
      "emoji": "🐛"
    },
    "docs": {
      "description": "Mudanças na documentação",
      "emoji": "📚"
    },
    "style": {
      "description": "Mudanças que não afetam o significado do código",
      "emoji": "💎"
    },
    "refactor": {
      "description": "Mudança de código que não corrige bug nem adiciona funcionalidade",
      "emoji": "📦"
    },
    "test": {
      "description": "Adição ou correção de testes",
      "emoji": "🚨"
    },
    "chore": {
      "description": "Mudanças no processo de build ou ferramentas auxiliares",
      "emoji": "🔧"
    }
  },
  "scopes": ["api", "ui", "core", "docs"],
  "rules": {
    "type-required": true,
    "type-case": "lower-case",
    "subject-required": true,
    "subject-case": "sentence-case",
    "subject-max-length": 50,
    "body-max-line-length": 72
  }
}

Tipos de Commit

Defina os tipos de commit permitidos no seu projeto. Cada tipo pode ter uma descrição e emoji opcional: 

{
  "types": {
    "feat": {
      "description": "Uma nova funcionalidade para o usuário",
      "emoji": "✨"
    },
    "fix": {
      "description": "Correção de bug para o usuário",
      "emoji": "🐛"
    },
    "perf": {
      "description": "Mudança de código que melhora performance",
      "emoji": "⚡"
    },
    "build": {
      "description": "Mudanças que afetam o sistema de build",
      "emoji": "🏗️"
    },
    "ci": {
      "description": "Mudanças nos arquivos e scripts de CI",
      "emoji": "👷"
    },
    "revert": {
      "description": "Reverte um commit anterior",
      "emoji": "⏪"
    }
  }
}

Escopos

Defina os escopos permitidos para organizar commits por área do projeto: 

{
  "scopes": [
    "auth",
    "api",
    "ui",
    "database",
    "config",
    "tests",
    "docs",
    "deps"
  ]
}

Se não especificado, qualquer escopo será aceito. Para desabilitar escopos completamente, defina como um array vazio: "scopes": []

Regras de Validação

Configure regras específicas para validação de commits:

Regra Tipo Padrão Descrição
type-required boolean true Tipo é obrigatório
type-case string lower-case Formato do tipo
subject-required boolean true Assunto é obrigatório
subject-case string sentence-case Formato do assunto
subject-max-length number 50 Tamanho máximo do assunto
body-max-line-length number 72 Tamanho máximo da linha do corpo

Configurações de UI

Controle o uso da tela alternada no modo interativo. Defina uiAltScreen para false para manter a UI interativa na tela principal do terminal. 

{
  "uiAltScreen": false
}

Nota: A flag --no-alt-screen no CLI tem precedência sobre configuração e variáveis de ambiente. Você também pode usar NO_ALT_SCREEN=1 para desabilitar a tela alternada via ambiente.

Integração com Git Hooks

Para validação automática durante commits, instale os Git hooks: 

commitzero install-hooks

Isso instalará hooks para:

  • commit-msg: Valida a mensagem do commit antes de confirmar
  • pre-commit: Executa validações antes do commit (opcional)

Para remover os hooks, use: commitzero uninstall-hooks

Comandos de Pre-commit

Execute comandos personalizados antes de cada commit (ex.: lint, testes): 

{
  "preCommitCommands": [
    "npm run lint",
    "npm test"
  ]
}

Timeout de Pre-commit

Defina o tempo máximo para executar comandos de pre-commit:

{
  "preCommitTimeout": "3m" // suporta ms, s, m
}

Precedência: flag da CLI -t/--timeout > variável COMMITZERO_PRE_COMMIT_TIMEOUT > configuração preCommitTimeout.

Nota: Se algum comando de pre-commit falhar, o commit será abortado.

Configuração Avançada

Configuração por Ambiente

Você pode ter diferentes configurações para diferentes ambientes: 

{
  "extends": ["@commitzero/config-conventional"],
  "environments": {
    "development": {
      "rules": {
        "subject-max-length": 100
      }
    },
    "production": {
      "rules": {
        "subject-max-length": 50,
        "body-required": true
      }
    }
  }
}

Configuração para Monorepo

Para projetos monorepo, você pode definir escopos baseados em pacotes: 

{
  "scopes": {
    "packages": true,
    "custom": ["shared", "tools", "docs"]
  },
  "rules": {
    "scope-required": true,
    "scope-case": "kebab-case"
  }
}
