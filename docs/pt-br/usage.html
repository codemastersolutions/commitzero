Guia de Uso

Aprenda como usar CommitZero no seu fluxo de trabalho diário para manter commits consistentes

Comandos CLI

Inicializar Configuração

Crie um arquivo de configuração inicial no seu projeto: 

commitzero init

Validar Mensagem de Commit

Valide uma mensagem de commit específica: 

# Validar mensagem diretamente
commitzero validate "feat: adicionar nova funcionalidade"

# Validar mensagem do último commit
commitzero validate --last

# Validar arquivo de mensagem
commitzero validate --file .git/COMMIT_EDITMSG

Instalar/Desinstalar Hooks

# Instalar hooks Git
commitzero install-hooks

# Desinstalar hooks Git
commitzero uninstall-hooks

# Verificar status dos hooks
commitzero hooks-status

Observação: uninstall-hooks e cleanup não removem scripts do package.json; os scripts commitzero, commitzero:install e commitzero:uninstall são preservados. A instalação respeita o core.hooksPath do Git e usa .git/hooks quando não definido; o CLI commitzero install-hooks configura core.hooksPath para .commitzero/hooks para que os hooks sejam versionados.

Limpeza manual se postuninstall não for executado

Se seu gerenciador de pacotes não executar postuninstall, remova as linhas entre # CommitZero managed block START e # CommitZero managed block END em:

  • .commitzero/hooks/commit-msg ou .git/hooks/commit-msg
  • .commitzero/hooks/prepare-commit-msg ou .git/hooks/prepare-commit-msg

Ou execute manualmente:

commitzero cleanup
# ou
npm run commitzero cleanup
# ou
pnpm commitzero cleanup
# ou
yarn commitzero cleanup

Ajuda

# Ver todos os comandos disponíveis
commitzero --help

# Ver ajuda para um comando específico
commitzero validate --help

Validação de Commits

Formato Válido

CommitZero segue o padrão Conventional Commits: 

tipo(escopo): descrição

corpo opcional

rodapé opcional

Exemplos Válidos

✅ Válidos

feat: adicionar login de usuário fix(api): corrigir erro de validação docs: atualizar README style(ui): melhorar espaçamento refactor: reorganizar estrutura

❌ Inválidos

adicionar login Fix: erro de validação feat: feat: Adicionar nova funcionalidade muito longa que excede o limite unknown: tipo inválido

Modo Interativo

Use o modo interativo para criar commits guiados: 

commitzero commit

Isso abrirá um assistente interativo que te guiará através de:

  • Seleção do tipo de commit
  • Escolha do escopo (se configurado)
  • Descrição do commit
  • Corpo opcional da mensagem
  • Rodapé opcional (breaking changes, issues)

Dicas de Navegação

  • Use as setas para cima/baixo para mover, Enter para confirmar
  • Pressione Esc para cancelar e voltar ao terminal
  • Digite para filtrar opções quando suportado

Opções de Exibição & Terminal

Desativar spinner/saída de progresso:

commitzero commit --progress-off

Executar sem tela alternativa (permanece no terminal principal):

commitzero commit --no-alt-screen

Você também pode definir NO_ALT_SCREEN=1 para desativar por padrão.

Variáveis de Ambiente

Configure o comportamento padrão via variáveis de ambiente. Flags da CLI têm precedência sobre configuração e variáveis de ambiente.

Idioma

Defina o idioma da interface para prompts e mensagens:

export COMMITZERO_LANG=pt

Valores suportados: en, pt, es.

Tela Alternativa

Desative o uso da tela alternativa globalmente:

export NO_ALT_SCREEN=1

Desfaça ou defina como 0 para habilitar. Para por comando, use --no-alt-screen.

Nota: Você também pode configurar padrões em .commitzero.json, incluindo uiAltScreen. Veja Configuração → Configurações de UI.

Integração com Git Hooks

Hook commit-msg

Após instalar os hooks, toda mensagem de commit será automaticamente validada: 

# Instalar hooks
commitzero install-hooks

# Agora todos os commits serão validados automaticamente
git commit -m "feat: nova funcionalidade"  # ✅ Válido
git commit -m "adicionar funcionalidade"   # ❌ Será rejeitado

Hook pre-commit

Opcionalmente, você pode configurar validações antes do commit: 

{
  "hooks": {
    "pre-commit": {
      "enabled": true,
      "commands": [
        "npm run lint",
        "npm run test"
      ]
    }
  }
}

Fluxo de Trabalho

  1. Fazer alterações no código
  2. Adicionar arquivos ao stage: git add .
  3. Fazer commit: git commit -m "feat: nova funcionalidade"
  4. Hook commit-msg valida a mensagem automaticamente
  5. Se válida, commit é criado; se inválida, é rejeitado com erro explicativo

Integração com Fluxo de Trabalho

CI/CD

Integre CommitZero no seu pipeline de CI/CD para validar commits em pull requests: 

# .github/workflows/commit-validation.yml
name: Validar Commits

on:
  pull_request:
    branches: [ main, develop ]

jobs:
  validate-commits:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
        with:
          fetch-depth: 0
      
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
      
      - name: Instalar CommitZero
        run: npm install -g commitzero
      
      - name: Validar commits
        run: |
          # Validar todos os commits no PR
          for commit in $(git rev-list origin/main..HEAD); do
            git show --format="%B" -s $commit | commitzero validate --stdin
          done

IDEs

Configure seu editor para usar CommitZero:

VS Code

Instale a extensão "Conventional Commits" e configure: 

{
  "conventionalCommits.scopes": ["api", "ui", "core", "docs"],
  "git.inputValidation": "always",
  "git.inputValidationLength": 50
}

Git GUI Tools

Para ferramentas como SourceTree, GitKraken, etc., os hooks Git funcionarão automaticamente.

Configuração para Equipe

Configuração Compartilhada

Mantenha a configuração no repositório para consistência da equipe: 

# Adicionar configuração ao repositório
git add .commitzero.json
git commit -m "chore: adicionar configuração CommitZero"

# Script de setup para novos desenvolvedores
echo "npm install -g commitzero && commitzero install-hooks" > setup-hooks.sh
chmod +x setup-hooks.sh

README da Equipe

Documente as convenções de commit no README: 

## Convenções de Commit

Este projeto usa [CommitZero](https://github.com/commitzero/commitzero) para validar commits.

### Setup
```bash
npm install -g commitzero
commitzero install-hooks
```

### Tipos Permitidos
- `feat`: Nova funcionalidade
- `fix`: Correção de bug
- `docs`: Documentação
- `style`: Formatação
- `refactor`: Refatoração
- `test`: Testes
- `chore`: Manutenção

### Exemplos
```
feat(auth): adicionar login OAuth
fix(api): corrigir validação de email
docs: atualizar guia de instalação
```

Solução de Problemas

🔧 Hooks não estão executando

Se os hooks Git não estão sendo executados: 

# Verificar se os hooks estão instalados
ls -la .git/hooks/

# Reinstalar hooks
commitzero uninstall-hooks
commitzero install-hooks

# Verificar permissões
chmod +x .git/hooks/commit-msg

📁 Configuração não encontrada

Se CommitZero não encontra a configuração: 

# Verificar se existe configuração
commitzero config --show

# Criar configuração se não existir
commitzero init

# Especificar arquivo de configuração
commitzero validate --config ./custom-config.json "feat: teste"

🚫 Erro de permissões

Para resolver problemas de permissão: 

# Instalar globalmente com sudo (se necessário)
sudo npm install -g commitzero

# Ou usar npx para evitar instalação global
npx commitzero validate "feat: teste"

# Configurar npm para não usar sudo
npm config set prefix ~/.npm-global
export PATH=~/.npm-global/bin:$PATH

Modo Debug

Use o modo debug para diagnosticar problemas: 

# Ativar modo debug
export DEBUG=commitzero:*

# Ou usar flag --debug
commitzero validate --debug "feat: teste"

# Ver configuração carregada
commitzero config --show --debug

Melhores Práticas

✅ Faça

  • • Use verbos no imperativo
  • • Seja específico e claro
  • • Mantenha o assunto conciso (<50 chars)
  • • Use o corpo para explicar o "porquê"
  • • Referencie issues quando relevante
  • • Use breaking changes quando necessário

❌ Não Faça

  • • Não use ponto final no assunto
  • • Não seja vago ("fix stuff")
  • • Não misture múltiplas mudanças
  • • Não use gerúndio ("adding", "fixing")
  • • Não ignore as convenções da equipe
  • • Não faça commits muito grandes
