Perguntas Frequentes

Encontre respostas rápidas para as dúvidas mais comuns sobre o CommitZero

CommitZero é uma ferramenta CLI que ajuda você a manter padrões consistentes de commit messages seguindo a especificação Conventional Commits. Ele valida suas mensagens de commit, oferece modo interativo para criação de commits e pode ser integrado ao seu fluxo de trabalho Git através de hooks.

Conventional Commits oferece vários benefícios:

  • Histórico claro: Commits padronizados facilitam a leitura do histórico
  • Automação: Permite geração automática de changelogs e versionamento semântico
  • Colaboração: Melhora a comunicação entre membros da equipe
  • Ferramentas: Integração com ferramentas de CI/CD e release automation

Sim! CommitZero é completamente gratuito e open source, licenciado sob MIT License. Você pode usar, modificar e distribuir livremente, tanto para projetos pessoais quanto comerciais.

CommitZero requer:

  • Node.js: Versão 16.0.0 ou superior
  • npm/yarn/pnpm: Qualquer gerenciador de pacotes Node.js
  • Git: Versão 2.0 ou superior
  • Sistema Operacional: Windows, macOS, ou Linux

Depende do seu caso de uso:

  • Global: Recomendado para uso pessoal em múltiplos projetos
  • Local: Melhor para projetos de equipe, garante que todos usem a mesma versão
  • NPX: Ideal para testes ou uso esporádico, sempre usa a versão mais recente
# Instalação global
npm update -g commitzero

# Instalação local
npm update commitzero

# Verificar versão atual
commitzero --version

Não é obrigatório. CommitZero funciona com configurações padrão que seguem a especificação Conventional Commits. Você só precisa criar um arquivo de configuração se quiser:

  • Personalizar tipos de commit permitidos
  • Definir escopos específicos para seu projeto
  • Ajustar regras de validação
  • Configurar mensagens de erro personalizadas

Sim! CommitZero procura por configuração na seguinte ordem:

  1. .commitzero.json no diretório atual
  2. .commitzero.js no diretório atual
  3. Seção commitzero no package.json
  4. Configuração global do usuário
  5. Configuração padrão

Adicione o arquivo .commitzero.json ao controle de versão do seu projeto. Todos os membros da equipe que clonarem o repositório usarão automaticamente a mesma configuração.

Você também pode instalar CommitZero como dependência de desenvolvimento no package.json para garantir que todos usem a mesma versão.

Se você receber "command not found", tente:

  1. Reiniciar o terminal
  2. Verificar se o diretório npm global está no PATH
  3. Usar npx commitzero em vez de commitzero
  4. Reinstalar: npm uninstall -g commitzero && npm install -g commitzero

Verifique se:

  • Os hooks foram instalados: commitzero install-hooks
  • Os arquivos de hook têm permissão de execução
  • Você está no diretório raiz do repositório Git
  • Não há conflitos com outros hooks existentes

No Windows, você pode encontrar erros de permissão. Soluções:

  • Execute o terminal como administrador
  • Use npm config set prefix para definir um diretório local
  • Instale usando --force se necessário
  • Considere usar WSL (Windows Subsystem for Linux)

Use a flag --debug para ver informações detalhadas: 

commitzero validate "feat: nova funcionalidade" --debug

Isso mostrará o processo de validação passo a passo, configuração carregada e possíveis erros.
