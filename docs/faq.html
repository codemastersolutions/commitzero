Frequently Asked Questions

Find answers to common questions about CommitZero installation, configuration, and usage.

General

CommitZero is a powerful CLI tool that helps enforce Conventional Commits standards in your Git workflow. It validates commit messages, provides interactive commit creation, and integrates seamlessly with Git hooks to ensure consistent commit message formatting across your team.

Conventional Commits provide several benefits:

  • Automated versioning: Generate semantic versions automatically
  • Clear history: Easily understand what changed in each commit
  • Better collaboration: Consistent format helps team communication
  • Automated changelogs: Generate release notes automatically
  • CI/CD integration: Trigger different workflows based on commit types

Yes! CommitZero is completely free and open-source, licensed under the MIT License. You can use it in personal projects, commercial applications, and contribute to its development on GitHub.

Installation

CommitZero requires:

  • Node.js: Version 16.0.0 or higher
  • npm: Version 7.0.0 or higher (or yarn/pnpm equivalent)
  • Git: Version 2.0.0 or higher
  • Operating System: Windows, macOS, or Linux

Yes, you can install CommitZero globally to use it across all your projects: 

npm install -g commitzero

However, we recommend installing it as a dev dependency in each project to ensure version consistency across team members.

To update CommitZero to the latest version: 

# For global installation
npm update -g commitzero

# For project installation
npm update commitzero

You can check your current version with commitzero --version

Configuration

No, CommitZero works out of the box with sensible defaults. However, creating a configuration file allows you to customize commit types, scopes, and validation rules to match your project's needs. Run commitzero init to create a basic configuration file.

Yes! You can define custom commit types in your configuration file: 

module.exports = {
  types: [
    'feat',     // New feature
    'fix',      // Bug fix
    'docs',     // Documentation
    'style',    // Code style
    'refactor', // Code refactoring
    'perf',     // Performance
    'test',     // Tests
    'chore',    // Maintenance
    'ui',       // UI changes (custom)
    'api',      // API changes (custom)
    'db'        // Database changes (custom)
  ]
};

Simply commit your commitzero.config.js file to your repository. When team members install CommitZero, it will automatically use the shared configuration. You can also add CommitZero as a dev dependency in your package.json to ensure everyone uses the same version.

Troubleshooting

If Git hooks aren't running, try these solutions:

  1. Ensure hooks are installed: commitzero install-hooks
  2. Check hook file permissions: chmod +x .git/hooks/commit-msg
  3. Verify you're in a Git repository: git status
  4. Check if hooks are being bypassed: avoid using git commit --no-verify
  5. Reinstall hooks if needed: commitzero uninstall-hooks && commitzero install-hooks

This usually means CommitZero isn't installed or not in your PATH:

  1. Check if it's installed: npm list -g commitzero
  2. Install globally: npm install -g commitzero
  3. Or use npx: npx commitzero --help
  4. Check your PATH includes npm global bin: npm config get prefix

Enable debug mode to get detailed information about what CommitZero is doing: 

# Enable debug mode
export DEBUG=commitzero:*

# Or for a single command
DEBUG=commitzero:* commitzero validate

# Use --verbose flag for more output
commitzero validate --verbose
