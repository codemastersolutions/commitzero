Contribuir a CommitZero

¡Damos la bienvenida a las contribuciones de la comunidad! Esta guía te ayudará a comenzar a contribuir a CommitZero.

Primeros Pasos

¿Primera vez contribuyendo?

Echa un vistazo a nuestros issues para principiantes para comenzar con tu primera contribución.

CommitZero es un proyecto de código abierto, y apreciamos todas las contribuciones, ya sean reportes de errores, solicitudes de características, mejoras en la documentación o contribuciones de código.

Antes de contribuir, por favor:

  • Lee nuestro Código de Conducta
  • Revisa los issues y pull requests existentes
  • Únete a nuestra comunidad de Discord para discusiones

Formas de Contribuir

Reportar Errores

¿Encontraste un error? Ayúdanos a mejorar reportándolo con información detallada y pasos de reproducción.

Sugerir Características

¿Tienes una idea para una nueva característica? Compártela con nosotros a través de solicitudes de características y discusiones.

Contribuir Código

Envía pull requests para corregir errores, agregar características o mejorar la funcionalidad existente.

Mejorar Documentación

Ayuda a hacer nuestra documentación más clara y completa para todos los usuarios.

Reportar Problemas

Al reportar problemas, por favor proporciona tanto detalle como sea posible para ayudarnos a entender y reproducir el problema.

Plantilla de Issue

Plantilla de Reporte de Error 
## Descripción del Error
Una descripción clara y concisa de cuál es el error.

## Pasos para Reproducir
1. Ve a '...'
2. Haz clic en '....'
3. Desplázate hacia abajo a '....'
4. Ver error

## Comportamiento Esperado
Una descripción clara y concisa de lo que esperabas que pasara.

## Comportamiento Actual
Una descripción clara y concisa de lo que realmente pasó.

## Entorno
- SO: [ej. macOS 12.0, Ubuntu 20.04, Windows 11]
- Versión de Node.js: [ej. 18.17.0]
- Versión de CommitZero: [ej. 1.2.3]
- Gestor de paquetes: [ej. npm 9.6.7, yarn 1.22.19]

## Contexto Adicional
Agrega cualquier otro contexto sobre el problema aquí, incluyendo:
- Archivos de configuración
- Mensajes de error o logs
- Capturas de pantalla (si aplica)

Configuración de Desarrollo

Sigue estos pasos para configurar tu entorno de desarrollo:

1. Fork y Clonar 
# Haz fork del repositorio en GitHub, luego clona tu fork
git clone https://github.com/TU_USUARIO/commitzero.git
cd commitzero

# Agrega el repositorio original como upstream
git remote add upstream https://github.com/commitzero/commitzero.git
2. Instalar Dependencias 
# Instalar dependencias
npm install

# O si prefieres yarn
yarn install
3. Ejecutar Pruebas 
# Ejecutar la suite de pruebas
npm test

# Ejecutar pruebas en modo watch
npm run test:watch

# Ejecutar pruebas con cobertura
npm run test:coverage
4. Construir y Probar Localmente 
# Construir el proyecto
npm run build

# Enlazar para pruebas locales
npm link

# Probar el CLI globalmente
commitzero --help

Estándares de Código

Seguimos estos estándares de código para mantener la calidad y consistencia del código:

Estilo de Código

  • • Usar TypeScript para todo código nuevo
  • • Seguir la configuración de ESLint
  • • Usar Prettier para formateo
  • • Escribir nombres de variables significativos
  • • Agregar comentarios JSDoc para funciones

Commits de Git

  • • Seguir Conventional Commits
  • • Usar tiempo presente ("add" no "added")
  • • Usar modo imperativo ("move" no "moves")
  • • Limitar primera línea a 72 caracteres
  • • Referenciar issues cuando aplique
Verificaciones Pre-commit 
# Ejecutar linting
npm run lint

# Corregir problemas de linting
npm run lint:fix

# Formatear código
npm run format

# Verificación de tipos
npm run type-check
