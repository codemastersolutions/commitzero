General

CommitZero es una herramienta CLI poderosa que ayuda a validar y hacer cumplir los estándares de Conventional Commits en tu flujo de trabajo de Git. Automatiza la validación de mensajes de commit, proporciona hooks de Git, y asegura la consistencia en todo tu equipo de desarrollo.

Conventional Commits proporciona varios beneficios:

  • Generación automática de changelogs: Crea changelogs basados en tipos de commit
  • Versionado semántico: Determina automáticamente los bumps de versión
  • Mejor colaboración: Mensajes de commit claros y consistentes
  • Herramientas mejoradas: Mejor integración con herramientas de CI/CD
  • Historial más limpio: Más fácil navegar y entender el historial del proyecto

¡Sí! CommitZero es completamente gratuito y de código abierto, licenciado bajo la Licencia MIT. Puedes usarlo en proyectos personales, aplicaciones comerciales, y contribuir a su desarrollo en GitHub.

Instalación

CommitZero requiere:

  • Node.js: Versión 16.0.0 o superior
  • npm: Versión 7.0.0 o superior (o equivalente yarn/pnpm)
  • Git: Versión 2.0.0 o superior
  • Sistema Operativo: Windows, macOS, o Linux

Sí, puedes instalar CommitZero globalmente para usarlo en todos tus proyectos: 

npm install -g commitzero

Sin embargo, recomendamos instalarlo como dependencia de desarrollo en cada proyecto para asegurar consistencia de versión entre los miembros del equipo.

Para actualizar CommitZero a la última versión: 

# Para instalación global
npm update -g commitzero

# Para instalación de proyecto
npm update commitzero

Puedes verificar tu versión actual con commitzero --version

Configuración

No, CommitZero funciona inmediatamente con valores predeterminados sensatos. Sin embargo, crear un archivo de configuración te permite personalizar tipos de commit, scopes, y reglas de validación para adaptarse a las necesidades de tu proyecto. Ejecuta commitzero init para crear un archivo de configuración básico.

¡Sí! Puedes definir tipos de commit personalizados en tu archivo de configuración: 

module.exports = {
  types: [
    'feat',     // Nueva funcionalidad
    'fix',      // Corrección de errores
    'docs',     // Documentación
    'style',    // Estilo de código
    'refactor', // Refactorización de código
    'perf',     // Rendimiento
    'test',     // Pruebas
    'chore',    // Mantenimiento
    'ui',       // Cambios de UI (personalizado)
    'api',      // Cambios de API (personalizado)
    'db'        // Cambios de base de datos (personalizado)
  ]
};

Simplemente haz commit de tu archivo commitzero.config.js a tu repositorio. Cuando los miembros del equipo instalen CommitZero, automáticamente usará la configuración compartida. También puedes agregar CommitZero como dependencia de desarrollo en tu package.json para asegurar que todos usen la misma versión.

Solución de Problemas

Si los hooks de Git no se ejecutan, prueba estas soluciones:

  1. Asegúrate de que los hooks estén instalados: commitzero install-hooks
  2. Verifica los permisos del archivo hook: chmod +x .git/hooks/commit-msg
  3. Verifica que estés en un repositorio Git: git status
  4. Verifica si los hooks están siendo omitidos: evita usar git commit --no-verify
  5. Reinstala los hooks si es necesario: commitzero uninstall-hooks && commitzero install-hooks

Esto usualmente significa que CommitZero no está instalado o no está en tu PATH:

  1. Verifica si está instalado: npm list -g commitzero
  2. Instala globalmente: npm install -g commitzero
  3. O usa npx: npx commitzero --help
  4. Verifica que tu PATH incluya el bin global de npm: npm config get prefix

Habilita el modo debug para obtener información detallada sobre lo que está haciendo CommitZero: 

# Habilitar modo debug
export DEBUG=commitzero:*

# O para un solo comando
DEBUG=commitzero:* commitzero validate

# Usa la bandera --verbose para más salida
commitzero validate --verbose
