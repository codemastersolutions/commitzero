Ejemplos de Uso Básico

✅ Mensajes de Commit Válidos

Adición de Funcionalidad 
feat: add user authentication system
Corrección de Error con Alcance 
fix(api): resolve memory leak in user service
Cambio Disruptivo 
feat!: migrate to new authentication API

BREAKING CHANGE: The old auth endpoints are no longer supported.
Users must update their API calls to use the new /v2/auth endpoints.
Actualización de Documentación 
docs(readme): update installation instructions

Add detailed steps for Windows users and troubleshooting section
for common installation issues.

Closes #45

❌ Mensajes de Commit Inválidos

Fixed stuff

Falta el tipo y una descripción adecuada

feat: Added new feature.

No debe terminar con punto y usar modo imperativo

FEAT: ADD USER AUTH

Debe estar en minúsculas

Ejemplos Específicos por Tipo de Proyecto

Proyecto Frontend (React/Vue/Angular)

commitzero.config.js 
module.exports = {
  types: [
    'feat',     // Nueva funcionalidad
    'fix',      // Corrección de error
    'docs',     // Documentación
    'style',    // Formato, punto y coma faltante, etc
    'refactor', // Cambio de código que no corrige error ni añade funcionalidad
    'perf',     // Mejora de rendimiento
    'test',     // Añadir pruebas faltantes
    'chore',    // Actualizar tareas de build, configuraciones del gestor de paquetes, etc
    'ui',       // Mejoras de UI/UX
    'a11y'      // Mejoras de accesibilidad
  ],
  scopes: [
    'components',
    'pages',
    'hooks',
    'utils',
    'styles',
    'assets',
    'config',
    'deps'
  ],
  rules: {
    'type-enum': ['error', 'always', [
      'feat', 'fix', 'docs', 'style', 'refactor',
      'perf', 'test', 'chore', 'ui', 'a11y'
    ]],
    'scope-enum': ['error', 'always', [
      'components', 'pages', 'hooks', 'utils',
      'styles', 'assets', 'config', 'deps'
    ]],
    'subject-case': ['error', 'always', 'lower-case'],
    'subject-max-length': ['error', 'always', 50]
  }
};

Ejemplos de Commits

feat(components): add loading spinner component
fix(hooks): resolve infinite loop in useEffect
ui(pages): improve mobile responsiveness
a11y(components): add ARIA labels to buttons
perf(utils): optimize image compression function

Ejemplos de Integración CI/CD

GitHub Actions

.github/workflows/commit-validation.yml 
name: Validate Commits

on:
  pull_request:
    branches: [main, develop]
  push:
    branches: [main, develop]

jobs:
  validate-commits:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v3
        with:
          fetch-depth: 0

      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Install CommitZero
        run: npm install -g commitzero

      - name: Validate commit messages
        run: |
          if [ "${{ github.event_name }}" = "pull_request" ]; then
            # Validate commits in PR
            commitzero validate --from=${{ github.event.pull_request.base.sha }} --to=${{ github.event.pull_request.head.sha }}
          else
            # Validate last commit on push
            commitzero validate --from=HEAD~1 --to=HEAD
          fi

      - name: Comment on PR (if validation fails)
        if: failure() && github.event_name == 'pull_request'
        uses: actions/github-script@v6
        with:
          script: |
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: '❌ **¡La validación de commits falló!**\n\nPor favor, asegúrate de que tus mensajes de commit sigan el estándar [Conventional Commits](https://conventionalcommits.org/).\n\nEjecuta `commitzero validate` localmente para verificar tus commits antes de hacer push.'
            })
