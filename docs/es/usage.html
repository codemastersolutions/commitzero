Guía de Uso

Aprende cómo usar los comandos CLI de CommitZero, validar commits e integrar con tu flujo de trabajo de desarrollo.

Comandos CLI

CommitZero proporciona varios comandos para ayudarte a gestionar la validación de commits y Git hooks.

Inicializar Proyecto

Configuración
commitzero init

Crea el archivo de configuración e instala Git hooks para validación automática.

Lo que hace:
  • Crea el archivo de configuración .commitzero.json
  • Instala los Git hooks pre-commit y commit-msg
  • Hace respaldo de hooks existentes si están presentes

Validar Mensaje de Commit

Validación
commitzero validate "feat: añadir autenticación de usuario"
commitzero validate --file mensaje-commit.txt

Valida un mensaje de commit contra las reglas de tu configuración.

Gestionar Git Hooks

Hooks

Instalar Hooks

commitzero install-hooks

Desinstalar Hooks

commitzero uninstall-hooks

Verificar Estado de Hooks

commitzero hooks-status

Instala o elimina manualmente Git hooks para validación de commits.

Nota: uninstall-hooks y cleanup no eliminan los scripts de package.json; los scripts commitzero, commitzero:install y commitzero:uninstall se conservan. La instalación respeta core.hooksPath de Git y usa .git/hooks cuando no está configurado; el CLI commitzero install-hooks configura core.hooksPath a .commitzero/hooks para que los hooks estén bajo control de versiones.

Limpieza manual si postuninstall no se ejecuta

Si tu gestor de paquetes no ejecuta postuninstall, elimina las líneas entre # CommitZero managed block START y # CommitZero managed block END en:

  • .commitzero/hooks/commit-msg o .git/hooks/commit-msg
  • .commitzero/hooks/prepare-commit-msg o .git/hooks/prepare-commit-msg

O ejecuta manualmente:

commitzero cleanup
# o
npm run commitzero cleanup
# o
pnpm commitzero cleanup
# o
yarn commitzero cleanup

Obtener Ayuda

Info
commitzero --help
commitzero --version

Muestra información de ayuda y detalles de la versión.

Validación de Commits

Comprende cómo CommitZero valida tus mensajes de commit y qué hace que un commit sea válido.

✅ Mensajes de Commit Válidos

feat: añadir sistema de autenticación de usuario
fix(api): resolver problema de timeout en endpoint de login
docs: actualizar instrucciones de instalación
feat!: cambiar formato de respuesta de API

❌ Mensajes de Commit Inválidos

añadida nueva funcionalidad

Falta prefijo de tipo

feat:

Falta descripción

feat: Añadir Nueva Funcionalidad

La descripción debe empezar con minúscula

desconocido: añadir algo

Tipo de commit inválido

Reglas de Validación

Requisitos de Formato

  • Ser específico y descriptivo
  • Usar scopes para contexto
  • Mantener commits atómicos
  • Incluir contexto en el cuerpo si es necesario

❌ No Hacer

  • Usar mensajes vagos como "fix stuff"
  • Mezclar múltiples cambios no relacionados
  • Usar tiempo pasado ("añadido", "corregido")
  • Exceder límites de longitud

    Modo Interactivo

    Usa el modo interactivo de CommitZero para crear mensajes de commit bien formateados con prompts guiados.

    Iniciar Modo Interactivo

    commitzero commit

    Prompts Interactivos

    ? Selecciona el tipo de cambio: (Usa las flechas)
    ❯ feat: Una nueva funcionalidad
    fix: Corrección de un error
    docs: Cambios solo en documentación
    ? ¿Cuál es el scope de este cambio? (opcional)
    api
    ? Escribe una descripción corta:
    añadir endpoint de autenticación de usuario

    Commit Generado

    feat(api): añadir endpoint de autenticación de usuario

    Consejos de Navegación

    • Usa flechas arriba/abajo para moverte; Enter para confirmar
    • Presiona Esc para cancelar y volver a la terminal
    • Escribe para filtrar opciones donde esté soportado

    Opciones de Visualización y Terminal

    Desactivar spinner/salida de progreso:

    commitzero commit --progress-off

    Ejecutar sin pantalla alternativa (mantenerse en la terminal principal):

    commitzero commit --no-alt-screen

    También puedes definir NO_ALT_SCREEN=1 para desactivarla por defecto.

    Variables de Entorno

    Configura el comportamiento predeterminado mediante variables de entorno. Las flags de CLI tienen prioridad sobre la configuración y el entorno.

    Idioma

    Define el idioma de la interfaz para prompts y mensajes:

    export COMMITZERO_LANG=es

    Valores soportados: en, pt, es.

    Pantalla Alternativa

    Desactivar el uso de pantalla alternativa globalmente:

    export NO_ALT_SCREEN=1

    Desactiva por defecto; desactiva o define 0 para habilitar. Para un solo comando, usa --no-alt-screen.

    Nota: También puedes configurar valores predeterminados en .commitzero.json, incluyendo uiAltScreen. Consulta Configuración → Ajustes de UI.

    Integración con Git Hooks

    CommitZero valida automáticamente los commits a través de Git hooks, asegurando consistencia en todo tu equipo.

    Hooks Instalados

    pre-commit

    Se ejecuta antes de crear el commit

    • • Ejecuta comandos pre-commit configurados
    • • Valida archivos staged
    • • Puede abortar el commit si falla

    commit-msg

    Se ejecuta después de escribir el mensaje

    • • Valida formato del mensaje
    • • Verifica tipos y scopes permitidos
    • • Aplica reglas de longitud

    Flujo de Trabajo con Hooks

    1

    Preparar Cambios

    Añade archivos con git add .

    2

    Validación Pre-commit

    Ejecuta comandos pre-commit configurados (linting, tests)

    3

    Mensaje de Commit

    Crea commit con git commit -m "mensaje"

    4

    Validación de Mensaje

    CommitZero valida formato y reglas del mensaje

    Integración de Flujo de Trabajo

    Integra CommitZero con tu flujo de trabajo de desarrollo existente y herramientas.

    Integración CI/CD

    Valida mensajes de commit en tu pipeline de CI/CD: 

    # Ejemplo de GitHub Actions
name: Validar Commits
on: [push, pull_request]
jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
        with:
          fetch-depth: 0
      - uses: actions/setup-node@v3
        with:
          node-version: '18'
      - run: npm install -g commitzero
      - run: commitzero validate-range HEAD~1..HEAD

    Solución de Problemas

    Problemas comunes y sus soluciones al usar CommitZero.

    Problemas Comunes

    Los hooks no se ejecutan

    Si los Git hooks no se ejecutan, verifica:

    • Los archivos de hook existen en .git/hooks/
    • Los archivos de hook son ejecutables (chmod +x)
    • No hay errores de sintaxis en los scripts de hook
    commitzero install-hooks --force

    Configuración no encontrada

    Si CommitZero no puede encontrar tu configuración:

    • Asegúrate de que .commitzero.json existe en la raíz del proyecto
    • Verifica que la sintaxis JSON es válida
    • Usa la bandera --config para especificar una ruta personalizada
    commitzero validate --config ./config-personalizado.json

    Errores de permisos denegados

    En sistemas Unix, podrías necesitar hacer los hooks ejecutables:

    chmod +x .git/hooks/pre-commit .git/hooks/commit-msg

    Modo de Depuración

    Habilita logging detallado para solucionar problemas:

    commitzero validate "feat: test" --verbose

    Esto mostrará pasos detallados de validación y carga de configuración.
